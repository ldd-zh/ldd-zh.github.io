linux设备驱动

Hello World模块

许多编程书籍都会以一个“Hello World”示例开始，用来展示最简单的程序。本书讨论的是内核模块，而不是普通程序；因此，对于急于开始的读者，以下代码就是一个完整的“Hello World”模块：

#include <linux/init.h>
#include <linux/module.h>
MODULE_LICENSE("Dual BSD/GPL");

static int hello_init(void)
{
    printk(KERN_ALERT "Hello, world\n");
    return 0;
}

static void hello_exit(void)
{
    printk(KERN_ALERT "Goodbye, cruel world\n");
}

module_init(hello_init);
module_exit(hello_exit);

这个模块定义了两个函数，一个是在模块加载到内核时调用的（hello_init），另一个是在模块卸载时调用的（hello_exit）。module_init 和 module_exit 宏用于标明这两个函数的作用。另一个特殊的宏（MODULE_LICENSE）用于告诉内核该模块采用的是自由许可证；如果没有这样的声明，内核在加载模块时会报错。

printk 函数是 Linux 内核中定义的，可以供模块使用；它的行为类似于标准 C 库中的 printf 函数。内核需要自己的打印函数，因为它是独立运行的，不依赖 C 库。模块可以调用 printk，因为在 insmod 加载模块后，模块与内核连接，并能够访问内核的公共符号（如函数和变量，详细内容见下一节）。(KERN_ALERT) 是消息的优先级。我们在这个模块中指定了较高的优先级，因为如果使用默认优先级，消息可能无法出现在有用的位置，这取决于你所使用的内核版本、klogd 守护进程的版本以及你的配置。你可以暂时忽略这个问题；我们将在第4章中进一步解释。

你可以使用 insmod 和 rmmod 工具来测试这个模块，如下所示。请注意，只有超级用户才能加载和卸载模块。

% make
make[1]: Entering directory `/usr/src/linux-2.6.10'
    CC [M] /home/ldd3/src/misc-modules/hello.o
    Building modules, stage 2.
    MODPOST
    CC /home/ldd3/src/misc-modules/hello.mod.o
    LD [M] /home/ldd3/src/misc-modules/hello.ko
make[1]: Leaving directory `/usr/src/linux-2.6.10'
% su
root# insmod ./hello.ko
Hello, world
root# rmmod hello
Goodbye, cruel world
root#

请再次注意，为了使上述命令序列能够正常工作，你必须有一个正确配置并构建好的内核源树，并且 Makefile 能够找到它（在示例中是 /usr/src/linux-2.6.10）。我们将在“编译与加载”一节中详细讨论模块的构建过程。

根据你的系统用于传递消息行的机制，输出可能会有所不同。特别地，前面的屏幕截图是从文本控制台获取的；如果你在窗口系统下运行终端仿真器并执行 insmod 和 rmmod，你将看不到任何信息。这些消息会被写入到系统日志文件中，比如 /var/log/messages（具体文件名在不同的 Linux 发行版中可能不同）。用于传递内核消息的机制将在第4章中讲解。

正如你所看到的，编写一个模块并不像你想象的那么困难——至少当模块不需要执行任何实际功能时是这样的。真正困难的部分在于理解你的设备以及如何最大化性能。我们将在本章深入探讨模块化，具体的设备相关问题会留到后面的章节讨论。
