linux设备驱动

模块参数

驱动程序需要知道的一些参数在不同系统之间可能会有所不同。这些参数可以从使用的设备编号（我们将在下一章中看到）到驱动程序操作方式的多个方面。例如，SCSI适配器的驱动程序通常有选项控制标记命令队列的使用，而集成设备电子（IDE）驱动程序允许用户控制DMA操作。如果你的驱动程序控制的是较旧的硬件，它可能还需要明确告知在哪里找到该硬件的I/O端口或I/O内存地址。内核通过允许驱动程序指定模块加载时可能更改的参数来支持这些需求。

这些参数值可以在加载时通过 insmod 或 modprobe 进行分配；后者还可以从其配置文件（/etc/modprobe.conf）中读取参数赋值。这些命令可以在命令行中指定多种类型的值。为了演示这一功能，假设对开头部分提到的“hello world”模块（称为 hellop）进行一个急需的增强。我们添加了两个参数：一个名为 howmany 的整数值和一个名为 whom 的字符串。这样，加载时，模块不仅会问候 whom 一次，而是问候 howmany 次。此时，可以通过以下命令行加载该模块：

insmod hellop howmany=10 whom="Mom"

加载后，hellop 将会说“Hello, Mom” 10 次。

然而，在 insmod 可以更改模块参数之前，模块必须使这些参数可用。参数通过 module_param 宏声明，module_param 宏定义在 moduleparam.h 中。module_param 宏接受三个参数：变量名、变量类型以及用于附加 sysfs 条目的权限掩码。该宏应放在函数外部，通常位于源文件的开头。因此，hellop 会按如下方式声明其参数，并使其对 insmod 可用：

static char *whom = "world";
static int howmany = 1;
module_param(howmany, int, S_IRUGO);
module_param(whom, charp, S_IRUGO);

模块参数支持多种类型：

  • bool：布尔值（真或假）（相关变量应为 int 类型）。
  • invbool：反转布尔值，将真值变为假，假值变为真。
  • charp：字符指针值。为用户提供的字符串分配内存，并相应地设置指针。
  • int、long、short、uint、ulong、ushort：各种长度的基本整数值，带有 u 前缀的版本用于无符号值。

还支持数组参数，其中值作为逗号分隔的列表提供。要声明一个数组参数，可以使用：

module_param_array(name, type, nump, perm);

其中 name 是数组（和参数）的名称，type 是数组元素的类型，nump 是一个整数变量，perm 是通常的权限值。如果在加载时设置数组参数，则 nump 会被设置为提供的值的数量。模块加载器会拒绝接受超过数组可容纳的值。

如果你确实需要一个不在上述列表中的类型，模块代码中有钩子允许你定义它们；有关如何做的详细信息，请参见 moduleparam.h。所有模块参数应给定一个默认值；insmod 只有在用户明确指定时才会更改该值。模块可以通过将参数与默认值进行比较来检查显式参数。

module_param 的最后一个字段是权限值；你应该使用 <linux/stat.h> 中找到的定义。此值控制谁可以访问 sysfs 中的模块参数表示。如果 perm 设置为 0，则根本没有 sysfs 条目；否则，它会在 /sys/module* 下出现，并带有给定的权限集。使用 S_IRUGO 让全世界可以读取该参数，但无法更改；S_IRUGO|S_IWUSR 允许 root 用户更改参数。请注意，如果通过 sysfs 更改参数，则模块看到的该参数的值会改变，但模块不会以其他方式得到通知。除非你准备好检测变化并做出相应反应，否则你可能不应该使模块参数可写。
