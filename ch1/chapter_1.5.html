linux设备驱动

版本编号

在开始编程之前，我们先简单介绍一下 Linux 使用的版本编号方案，并说明本书所涉及的版本。

首先需要注意的是，每个 Linux 系统中使用的软件包都有自己的版本号，且这些软件包之间常常存在依赖关系：你可能需要某个特定版本的软件包才能运行另一个特定版本的软件包。Linux 发行版的开发者通常会处理好这些软件包之间的兼容问题，使用预打包发行版的用户不需要关心版本号。而那些自行替换和升级系统软件的用户，则需要自行管理这些依赖关系。幸运的是，几乎所有现代的发行版都支持通过检查软件包间的依赖关系来升级单个软件包；发行版的包管理器通常在依赖关系满足之前，不会允许软件包升级。

为了运行本书中的示例，你不需要特定版本的工具，只要满足 2.6 内核的要求，任何现代的 Linux 发行版都可以运行我们的示例。我们不会详细列出具体的要求，因为如果你在使用过程中遇到问题，Documentation/Changes 文件是查看这些信息的最佳来源。

对于内核而言，偶数版本（例如 2.6.x）是稳定版本，适用于广泛分发。与此相对，奇数版本（如 2.7.x）是开发快照，存在时间较短；最新的奇数版本代表了当前开发的状态，但几天后可能就会过时。

本书涵盖的是 2.6 版本的内核。我们的重点是展示 2.6.10（本书撰写时的当前版本）中提供给设备驱动开发者的所有功能。本书的这一版本没有涉及内核的早期版本。如果你对早期版本感兴趣，第二版详细涵盖了 2.0 到 2.4 的版本，仍然可以在 http://lwn.net/Kernel/LDD2/ 在线查看。

内核开发者应当知道，2.6 版本的开发流程发生了变化。2.6 系列开始接受那些以前被认为对“稳定”内核来说过大的变动。这意味着内核的内部编程接口可能发生变化，从而可能使本书中的部分内容过时。因此，虽然本书附带的示例代码在 2.6.10 上能够正常工作，但某些模块在早期版本中无法编译。想要跟进内核编程变化的开发者，建议加入邮件列表，并利用参考书目中列出的网站。还可以访问 http://lwn.net/Articles/2.6-kernel-api/，该页面提供了自本书出版以来发生的 API 变化信息。

本书没有专门讨论奇数版本的内核。一般用户无需运行开发版内核。那些开发新特性并进行实验的开发者，通常会使用最新的开发版本，他们不断升级以获取 bug 修复和新特性实现。然而需要注意的是，实验版内核并没有保证其稳定性，如果由于非当前的奇数版本内核中的 bug 导致问题，没人会提供帮助。通常使用奇数版本内核的人技术足够高，他们能自己调试代码而不需要依赖教科书，这也是我们没有讨论开发版内核的原因之一。

Linux 的另一个特点是它是一个平台独立的操作系统，不仅仅是“PC 克隆的 Unix 克隆”——它目前支持约 20 种架构。本书尽可能保持平台独立，所有代码示例至少在 x86 和 x86-64 平台上进行了测试。由于这些代码在 32 位和 64 位处理器上均已测试过，它应该能够在所有其他平台上编译并运行。如你所料，依赖特定硬件的代码示例并不适用于所有支持的平台，但这一点总会在源代码中明确标明。
